## [Unreleased]

### Security
- **Webhook admin API:** **`GET` / `POST` / `PATCH /api/webhooks`** responses no longer include the signing **`secret`** field (type **`WebhookEndpointPublic`** in **`shared/schema.ts`**; redaction in **`server/webhook-endpoint-public.ts`** and **`server/storage.ts`**). ### Changed
- **Storage:** **`updateWebhookEndpoint`** builds Drizzle **`.set()`** with **`WebhookEndpointUpdateSet`** — **`Partial<Pick<typeof webhookEndpoints.$inferInsert, "url" | "secret" | "eventTypes" | "enabled">> & { updatedAt: Date }`** — instead of **`Record<string, unknown>`** (**`server/storage.ts`**).
- **Webhook admin API:** **`PATCH`** / **`DELETE /api/webhooks/:id`** validate **`id`** with **`parsePositiveIntPathParam`** (**`server/validation/query-params.ts`**, Zod **`coerce` + `int` + `positive`**). Non-integers, zero, and negatives return **`400`** **`{ message: "Invalid id" }`**. Tests: **`server/tests/query-params.test.ts`**.
- **Docs:** **`docs/HARDENING-FOLLOWUPS.md`** §1 — **Risk** / **Code** lines updated so they no longer imply delivery is **`fetch(storedUrl)`**; they describe **`resolveWebhookOutboundConnect`**, pinned **`connectAddress`**, and **`webhookHttpDelivery.send`** (**`webhook-delivery-http.ts`**), with webhook CRUD under **`server/routes/webhook-routes.ts`**.
- **Docs:** **`docs/HARDENING-FOLLOWUPS.md`** §1 — operator note that HTTPS webhook URLs with a **literal IP** host use that IP for TLS **SNI** (`tlsServerName` / Node `servername`); prefer hostnames when the receiver expects a DNS name on the certificate. **`Readme.txt`** DOCUMENTATION index links to section 1 for this caveat.
- **[Breaking]** **`summarizeWebhookTargetForOps`** now returns **`{ hostname, port, pathFingerprint }`** instead of **`{ host, pathFingerprint }`**. Ops event payloads for **`job.webhook_delivery_dead`** expose **`port`** explicitly when it is not the default for **`http`/`https`**; default ports are represented as **`port: null`**. Historical **`ops_events`** rows may still use the old **`host`** field.
- **Webhook delivery (DNS TOCTOU mitigation):** Delivery no longer uses **`fetch(storedUrl)`**. **`resolveWebhookOutboundConnect`** (`server/webhook-url-policy.ts`) re-validates host policy and performs a fresh **`dns.lookup`** for hostnames, then **`webhookHttpDelivery.send`** (`server/webhook-delivery-http.ts`) opens **`http`/`https`** to the **pinned** address with **`Host`** and TLS **SNI** from the original URL. **`validateWebhookOutboundUrl`** refactored to share parsing/host checks. Tests: **`server/tests/webhooks.test.ts`**, **`server/tests/webhook-url-policy.test.ts`**.
- **Server routes refactor (maintainability, no API changes):** Shared **`server/route-middleware.ts`** (includes **`authEnvFlags`**, **`siteAccessJson`**), **`server/inventory-list-context.ts`**, **`server/inventory-bulk-undo-helpers.ts`** + **`server/inventory-bulk-undo.ts`**; **`server/upload-config.ts`** (multer targets, HEIC normalization, thumbnail rate-limit state); domain modules under **`server/routes/`**: **`auth-routes`**, **`user-routes`**, **`history-routes`**, **`shared-notes-routes`**, **`employee-docs-routes`**, **`company-routes`**, **`upload-routes`**, **`inventory-export-routes`**, **`inventory-item-routes`** (list/CRUD/assign/return/attachments), plus earlier **`webhook-routes`**, **`maintenance-routes`**, **`compliance-routes`**, **`reports-ops-routes`**, **`inventory-bulk-routes`**. **`server/routes.ts`** is a thin **`registerRoutes`** dispatcher (~**189** lines including CSRF, **`/api/inventory/filters`**, **`/api/sites`**, and startup seed). Shared HTTP errors for assign/return: **`server/http-status-error.ts`**. **Shared query parsing:** **`server/validation/query-params.ts`** (Zod) for inventory list + history pagination, webhook deliveries **`limit`**, and **`GET /api/ops-health/events`** **`limit`/`severity`** — **`docs/HARDENING-FOLLOWUPS.md`** §8.
- **Site RBAC (hardening):** Template `capabilities` from the DB are normalized to known site capability strings (`shared/site-rbac.ts`: `SITE_KNOWN_CAPABILITY_SET`, `parseSiteCapabilityStringsFromJsonb`). Unknown keys are ignored; if a user with explicit site grants would otherwise have **no** known capabilities, effective caps fall back to the user’s **global** role (same as pre-grant behavior for that role) so corrupt JSON cannot silently lock them out; non-production **`console.warn`** on **every** such fallback (including empty **`[]`** templates), with **`unknownSamples`** and **`hadAnyRawCapabilityStrings`** in the detail object. Tests: **`server/tests/site-rbac.test.ts`**, **`server/tests/site-rbac-capabilities.test.ts`**.
- **Executive summary (hardening):** Ops block (`getOpsSummary` / `reliability`) is included only when **`role === "editor"` or `role === "admin"`** (`server/load-executive-summary.ts`). **`GET /api/reports/executive-summary/pdf`** builds the PDF in memory, then sends it — failures yield **`500` JSON** if headers were not sent yet. Tests: **`server/tests/executive-summary.test.ts`**.
- **`GET /api/webhooks/deliveries`:** **Editor** responses use **`payload: null`** and **`payloadRedacted: true`**; **admin** keeps full **`payload`**. Mapper in **`server/webhook-deliveries.ts`**; tests **`server/tests/webhook-deliveries-redaction.test.ts`**.
- **`GET /api/sites`:** Comment documents legacy (no grant rows → all sites) vs grant-filtered list when site RBAC enforces (**`DECISIONS.md`** 2026-04-08).

Otherwise **`400`** with **`code: invalid_site_id`**. Tests: **`server/tests/inventory-list-context-siteid.test.ts`**, **`server/tests/sites-scoping.test.ts`**, **`server/tests/executive-summary.test.ts`**.
- **Dashboard / Resumen UI:** shared `KpiCard` (corner accent blob, theme colors), EcoOcéano-style refresh on Overview and toggleable inventory analytics (`rounded-2xl` cards, donut pies with legend, softer borders and spacing); analytics panel wrapper on `/inventory` updated to match.

### Security
- **Site RBAC:** `capsForGlobalRole` (**`shared/site-rbac.ts`**) no longer grants **`ALL_SITE_CAPABILITIES`** for unexpected `users.role` text; unknown values fail closed to **`inventory:read`** (non-production warning). Aligns with strict role gates elsewhere (e.g. executive summary). Tests: **`server/tests/site-rbac-capabilities.test.ts`**.
- **Outbound webhooks (SSRF mitigation):** `server/webhook-url-policy.ts` — only **`http`/`https`**, no URL-embedded credentials, blocks literal private/reserved IPs and sensitive hostnames, and (unless **`WEBHOOK_ALLOW_PRIVATE_TARGETS`**) rejects hostnames whose DNS **`A`/`AAAA`** include any private/reserved address. Enforced on **`POST`/`PATCH` `/api/webhooks`** and again at delivery (policy + second lookup for pin); violations at delivery mark outbox **`dead`** and emit **`job.webhook_delivery_dead`** with **`reason: url_policy`**. Pinned **`http`/`https`** transport reduces DNS time-of-check vs time-of-use drift vs **`fetch(hostname)`** alone (residual race between last lookup and socket connect is minimal). Tests: **`server/tests/webhook-url-policy.test.ts`**. See **`DECISIONS.md`** (2026-04-09, 2026-04-10) and **`docs/HARDENING-FOLLOWUPS.md`** §1.
- **`job.webhook_delivery_dead` ops payload privacy:** Raw **`url`** removed; payloads use **`webhookTarget: { hostname, port, pathFingerprint }`** via **`summarizeWebhookTargetForOps`** (**`server/webhook-ops-url.ts`**). Full URL remains on **`webhook_endpoints`** for admins. Tests: **`server/webhook-ops-url.test.ts`**, **`server/tests/webhooks.test.ts`**.
- **Outbound webhooks — IPv6 literals:** **`isPrivateOrReservedIpv6String`** extended in the same module: **RFC 3849** documentation (**`2001:db8::/32`**), **RFC 3879** deprecated site-local (**`fec0::/10`**), **RFC 6666** discard (**`100::/64`** canonical form), and **`ff00::/8`** multicast, with an explicit best-effort note in code (unusual **`::`** compressions may not all match). Tests: **`server/tests/webhook-url-policy.test.ts`**.
